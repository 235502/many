# Claude 学习仓库说明文档

## 📖 语言要求
永远只使用中文与用户对话，所有回复解释，提问，总结等均必须使用中文。

## 📖 仓库用途
这是一个交互式学习仓库，每个课题是一个独立的文件夹。

## 🗂️ 仓库结构
```
学习内容名称/
├── 01.md     # 基础介绍
```

## 🔄 交互式学习工作流

### 1. 文章组织方式
- 每个课题文件夹内的文章按序号命名：`01.md`、`02.md`、`03.md`...
- 序号顺序决定了学习路径的递进关系

### 2. 学习流程
1. **阅读阶段**：用户按顺序阅读文章
2. **反馈阶段**：用户在文章末尾或文件夹内写下：
   - 问题（疑问、困惑点）
   - 感悟（理解、心得）
   - 反馈（建议、改进意见）

### 3. 自适应学习机制
- 生成下一篇文章时，会先阅读用户在前一篇文章中的所有反馈
- 基于用户的理解程度和兴趣方向，动态调整内容深度和方向
- 形成个性化的学习梯度，确保内容既不过于简单，也不过于跳跃

## 📝 文章格式约定

### 基础结构
```markdown
# 文章标题

## 🎯 学习目标
- 明确本篇文章要掌握的内容

## 📚 核心内容
- 知识点的详细讲解
- 代码示例（如适用）
- 实践练习

## 💭 思考与讨论
- 引导性问题
- 拓展思考方向

## 📝 你的反馈
<!-- 用户在此写下问题、感悟或反馈 -->
```

### 内容要求
- 使用清晰易懂的语言解释复杂概念
- 提供实际代码示例和运行结果
- 包含适量的练习和思考题
- 保持适中的内容密度，避免信息过载

## 🤝 协作约定

### 对 Claude 的要求
1. 认真阅读和理解用户的历史反馈
2. 根据用户的掌握程度调整内容难度
3. 及时回答用户提出的问题
4. 提供有针对性的练习和实践建议

### 对用户的要求
1. 认真阅读每篇文章
2. 积极思考和提问
3. 及时记录学习心得和困惑
4. 完成推荐的练习和实践
